Для первого цикла

n статически объявлена как Number
компилятор видит только этот базовый тип
Поэтому всегда вызывает наименее специфичный метод:

public void printNum(Number n)
    {
        System.out.println("Number=" + n);
    }

Для второго цикла

Компилятор видит фактический точный тип 
Использует правило приоритета перегрузки, чтобы выбрать наиболее подходящий метод
